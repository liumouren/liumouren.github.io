使用hexo在github上搭建自己的静态博客

万事开头难,折腾了半天,总算是把这个博客给搭起来了.其实过程也并没有想象的那么难.所以在这里我也跟大家分享一下搭建此博客的过程以及遇到过的一些问题.在开始之前呢,先还是来了解一下搭建前需要了解的知识和准备工作.

hexo是什么?

Hexo 是一款基于node 的静态博客网站生成器,是由台湾的一名大学生tommy351所写.相比其他的静态网页生成器而言,有着生成静态网页最快，插件丰富的优势.用官方的话来说hexo就是一个快速、简洁且高效的博客框架,Hexo 使用 Markdown(或其他渲染引擎)解析文章,在几秒内,即可利用靓丽的主题生成静态网页。

官网地址:https://hexo.io

中文网:https://hexo.io/zh-cn

hexo的内容大致就说这么多了,具体细节大家可以查看中文网的文档去进行操作,文档的说明还是比较全面和详细的,按照官方的步骤去执行的话,是可以在很短时间内搭建属于自己的博客的.本文主要详细说明一下如何把安装后的博客发布到github上去.然而这些在hexo的官网是没有的.

github与git

git,一个非常强大的版本管理工具.Github则是一个基于Git的日益流行的开源项目托管库.Git与svn的最大区别是,它的使用流程不需要联机，可以先将对代码的修改,评论,保存在本机.等上网之后,再实时推送过去。同时它创建分支与合并分支更容易,推送速度也更快,配合Github提交需求也更容易.

实际上,我们只需在自己电脑安装上github的客户端即可把我们的博客发布到github pages上去了.至于为何用github pages的话,我想这都是毋庸置疑的.github pages有着300M的免费空间,资料自己保存管理,这样咱才放心嘛,对不对.最主要是github是趋势,上面也有很多大牛.总之好处多多吧 - -

github for windows 下载地址:https://windows.github.com

github for mac 下载地址:https://mac.github.com

安装完成以后,运行github,登录你的github帐号即可.没有帐号的话,大家自行去https://github.com注册帐号.如下图(点击sign up注册即可):

注册的过程就不详解了,都是比较简单的输入帐号啊密码邮箱什么的就可以了.注册完成以后我们就开始要建立自己的仓库了,如下图:

图中所示是因为我这个账号之前已经有建过一个仓库了,新账号注册是没有的,所以我们点击右上角的 + 号新建一个repository或者点击下方绿色按钮也可.如下图所示:

我们需要为自己的仓库指定一个repository name,这里要注意了,这个name的格式必须为 your username.github.io,如图所示,把xxxxxx替换成你的username即可.然后点击create repository.(由于我们是用的免费版,所以不能创建私有的仓库,如有需要,大家可自行购买收费版).创建好了自己的仓库以后,我们还需要做一个很重要的步骤,就是发布github pages.请看下图:

可以看到我新建的仓库里面已经有很多东西了,没错,这个就是我们通过hexo生成的静态博客,不过新帐号是没有的哟!我们点击右侧的settings菜单,进入设置界面,往下滑动到以下位置:

然后我们点击Launch automatic page generator去发布自己的page.注意(如果你想删除自己的仓库的话,点击下面的delete this repository即可),点击后会跳到一个layout界面去,意思就是设置你自己站点的布局,这里我们直接跳过,点击下面的continue to layouts继续,最后到达最终的发布页面.

这里提醒大家的是,上面虽然有各种选择的样式,但是我们的最终目的是搭建hexo,而不是创建自己的github pages,所以这里随便选一个主题,然后点击 publish page即可.至此,github这边的工作已经基本完成了.发布以后我们就能通过 http://liumouren.github.io去访问自己的站点了.

好了,到此基本上也已经讲完了.大致的总结一下吧.

  1. 首先我们需要安装在你本地电脑安装git或者github客户端
  2. 到github官网注册一个账号,并新建一个仓库发布
  3. 根据hexo官方文档介绍,生成本地静态博客
  4. 上传你的博客
